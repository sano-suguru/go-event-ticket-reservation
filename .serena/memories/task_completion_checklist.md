# タスク完了時チェックリスト

タスクを完了したら、以下を確認・実行してください。

## 1. コード品質チェック
```bash
make lint
```
golangci-lintでエラーがないこと

## 2. テスト実行
```bash
make test
```
全テストがパスすること（-race -cover付き）

## 3. TDDワークフロー（必須）
1. 失敗するテストを1つ書く（Red）
2. テストをパスする最小限のコードを実装（Green）
3. テストをグリーンに保ちながらリファクタリング（Refactor）

## 4. テストカバレッジ目標
| レイヤー | 目標カバレッジ |
|---------|---------------|
| Domain | 100% |
| Application | 89%以上 |
| Handler | 91%以上 |

## 5. コードレビュー前チェック
- [ ] `make lint` がパス
- [ ] `make test` がパス
- [ ] 新規コードにテストがある
- [ ] ドメインエラーは `errors.go` に定義
- [ ] トランザクションは適切にロールバック設定
- [ ] 日本語コメントが適切に記載

## 6. 統合テスト（大きな変更時）
```bash
make docker-up
make test-integration
```

## 7. ビルド確認
```bash
make build
```
コンパイルエラーがないこと
